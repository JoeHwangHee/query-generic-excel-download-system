# 사용자 매뉴얼 — 쿼리 기반 엑셀 다운로드

이 문서는 **쿼리 기반 제네릭 엑셀 다운로드 시스템**을 처음 사용하는 사용자를 위한
조작 안내서입니다. 설치·실행부터 DB 등록, 쿼리 저장·실행, 결과 확인, 자동 배치까지
실제 화면 흐름 순서대로 설명합니다.

> 개발/빌드 관점의 기술 개요는 [README.md](../README.md)를 참고하세요.

---

## 1. 시작하기 전에

| 항목 | 내용 |
|---|---|
| 운영체제 | Windows (결과물이 `C:\Temp\query-excel-download\`에 저장됨) |
| 실행 환경 | JDK 25 (fat jar 실행 시) |
| 대상 DB | MySQL / MariaDB |
| 출력 형식 | `.xlsx` (Excel) |

> ⚠️ 이 도구는 **SELECT 쿼리만** 실행할 수 있습니다. INSERT/UPDATE/DELETE 등
> 데이터를 변경하는 쿼리는 저장 단계에서 거부됩니다.

---

## 2. 프로그램 실행

빌드된 실행 파일(fat jar)이 있다면 명령 프롬프트에서 다음과 같이 실행합니다.

```bash
java -jar query-excel-download.jar
```

> 직접 빌드해야 한다면 [README.md](../README.md)의 *빌드 / 실행* 절을 참고하세요.

실행하면 **메인 화면**이 나타납니다. 저장된 쿼리 목록과 함께 아래 버튼이 있습니다.

| 버튼 | 기능 |
|---|---|
| **새 쿼리** | 새 쿼리를 작성·저장 |
| **실행** | 목록에서 선택한 쿼리를 실행하고 엑셀로 내려받기 |
| **삭제** | 선택한 쿼리를 삭제 (확인 후 삭제) |

처음 실행하면 목록이 비어 있습니다. 아래 순서대로 **DB를 등록 → 쿼리를 저장 →
쿼리를 실행**하면 됩니다.

---

## 3. DB 접속 정보 등록

쿼리를 저장하려면 먼저 실행할 대상 DB를 등록해야 합니다. DB 등록은
**[새 쿼리]** 화면 안의 **[DB 추가]** 버튼에서 진행합니다.

1. 메인 화면에서 **[새 쿼리]** 클릭 → 쿼리 저장 화면이 열립니다.
2. **실행 DB** 줄의 **[DB 추가]** 버튼 클릭 → DB 추가 창이 열립니다.
3. 다음 항목을 입력합니다.

   | 입력 항목 | 설명 | 필수 |
   |---|---|---|
   | **설정명** | 목록에서 식별할 이름 (예: `운영DB`, `사내통계`) | ✅ |
   | **JDBC URL** | 예: `jdbc:mysql://localhost:3306/mydb` | ✅ |
   | **사용자** | DB 계정 ID | |
   | **비밀번호** | DB 계정 비밀번호 | |
   | **드라이버** | 기본값이 자동 입력됨 (보통 그대로 사용) | ✅ |

4. **[연결 테스트]** 를 눌러 접속이 되는지 먼저 확인합니다.
   - 성공: `접속 성공` 메시지
   - 실패: `접속 실패: <원인>` 메시지 → URL·계정·네트워크를 점검하세요.
5. **[저장]** 을 누르면 등록되고, 방금 등록한 설정명이 **실행 DB** 콤보에
   자동 선택됩니다.

> 같은 설정명으로 다시 저장하면 기존 접속 정보가 새 값으로 교체됩니다.

## 4. 쿼리 저장

쿼리 저장 화면(**[새 쿼리]**)에서 다음을 입력합니다.

### 4-1. 기본 입력

1. **쿼리명** — 목록에 표시되고 결과 파일명에도 사용됩니다. (예: `일별매출`)
2. **SQL** — 실행할 SELECT 문을 입력합니다.
   - 변수가 들어갈 자리에는 물음표 **`?`** 를 사용합니다.
   - 예시:
     ```sql
     SELECT order_date, customer, amount
     FROM orders
     WHERE order_date = ? AND region = ?
     ```
3. **배치조건** — 자동 실행 주기를 선택합니다. (5절 참고, 자동 실행이 필요 없으면 `NONE`)
4. **실행 DB** — 3절에서 등록한 DB를 선택합니다.

### 4-2. 파라미터(`?`) 형식 지정

1. SQL 입력 후 **[? 파라미터 인식]** 버튼을 누릅니다.
   → SQL 안의 `?` 개수만큼 형식 지정 행(`?#1`, `?#2` …)이 생성됩니다.
2. 각 `?`에 대해 형식을 지정합니다.

   | 형식 | 입력 방식 (실행 시) |
   |---|---|
   | **TEXT** | 문자 입력칸 |
   | **NUMBER** | 숫자 입력칸 |
   | **DATE** | 달력 선택기 (날짜를 클릭해 선택) |

3. 형식이 **DATE** 인 경우 **날짜포맷** 을 함께 선택합니다.

   | 날짜포맷 | 쿼리에 바인딩되는 형태 |
   |---|---|
   | `yyyyMMdd` | 예: `20260615` |
   | `yyyy-MM-dd` | 예: `2026-06-15` |

   > 날짜포맷은 DB 컬럼이 날짜 문자열을 어떤 형태로 저장하는지에 맞춰 선택하세요.

### 4-3. 저장

- **[저장]** 클릭 → `저장되었습니다` 메시지 후 메인 화면 목록에 추가됩니다.
- 저장이 거부되는 경우와 대처:

  | 메시지 | 원인 / 대처 |
  |---|---|
  | `쿼리명과 SQL을 입력하세요.` | 쿼리명 또는 SQL이 비어 있음 |
  | `실행 DB를 선택하거나 'DB 추가'로 등록하세요.` | 등록된 DB가 없음 → 3절 수행 |
  | `'? 파라미터 인식'을 다시 눌러 형식을 지정하세요.` | SQL의 `?` 개수와 지정된 형식 행 수가 불일치 → 버튼 재클릭 |
  | `검증 오류` (SELECT 아님 / `?` 카운트 오류) | SELECT 문만 허용, SQL 수정 |

---

## 5. 쿼리 실행 및 다운로드

1. 메인 화면 목록에서 실행할 쿼리를 선택하고 **[실행]** 을 누릅니다.
2. 실행 화면에는 저장 시 지정한 형식대로 **입력칸이 자동 생성**됩니다.
   - TEXT/NUMBER → 입력칸에 값 입력
   - DATE → 달력에서 날짜 선택 (선택하지 않으면 `날짜를 선택하세요.` 안내)
3. **[실행 및 다운로드]** 클릭 → 지정된 DB에 접속해 백그라운드로 쿼리를 실행합니다.
4. 완료되면 다음 정보가 표시됩니다.
   ```
   완료: <건수>행
   C:\Temp\query-excel-download\<쿼리명>_<일시>.xlsx
   C:\Temp\query-excel-download\<쿼리명>_<일시>.txt
   ```
5. 실패 시 `실행 실패: <원인>` 메시지가 표시됩니다. (DB 접속 불가, SQL 오류 등)

## 6. 결과물 확인

모든 결과물은 다음 폴더에 자동 저장됩니다.

```
C:\Temp\query-excel-download\
```

| 파일 | 설명 |
|---|---|
| `쿼리명_yyyyMMdd_HHmmss.xlsx` | 쿼리 결과 엑셀. 결과 컬럼이 그대로 헤더가 됨 (컬럼 순서 보존) |
| `쿼리명_yyyyMMdd_HHmmss.txt` | 실제 실행된 SQL (변수가 채워진 상태) |
| `run-history.json` | 실행 이력 (배치 중복 실행 방지용, 직접 수정 불필요) |

> 엑셀 파일명과 SQL 텍스트 파일명은 동일한 시각으로 짝을 이루므로, 어떤 조건으로
> 뽑은 결과인지 `.txt` 로 추적할 수 있습니다.

## 7. 자동 배치 (기동 시 실행)

쿼리 저장 시 **배치조건** 을 지정하면, 프로그램을 켤 때 해당 날짜에 해당하는
쿼리를 자동으로 안내·실행합니다.

### 배치조건 종류

| 배치조건 | 의미 |
|---|---|
| `NONE` | 자동 실행 없음 |
| `EVERY_MONDAY` | 매주 월요일 |
| `FIRST_MONDAY` | 매월 첫째 주 월요일 |
| `DAY_1` | 매월 1일 |
| `DAY_26` | 매월 26일 |
| `LAST_WEEK_MONDAY` | 매월 마지막 주 월요일 |

### 동작 방식

1. 프로그램을 켜면 오늘(또는 미실행으로 넘어온 익일)이 배치일인 쿼리를 찾습니다.
2. 대상이 있으면 **배치 알림** 창으로 대상 목록을 안내합니다.
3. 안내 후 대상 쿼리의 **실행 화면이 하나씩 순서대로** 표시됩니다.
   → 5절과 동일하게 파라미터를 입력하고 **[실행 및 다운로드]** 하면 됩니다.
4. 이미 실행한 배치는 `run-history.json` 으로 중복 실행을 방지합니다.

## 8. 자주 묻는 질문 (FAQ)

**Q. INSERT/UPDATE 쿼리도 저장할 수 있나요?**
A. 아니요. 보안상 SELECT 문만 허용됩니다.

**Q. 결과를 `.xls`(구버전)로 받을 수 있나요?**
A. 아니요. 메모리 효율을 위해 `.xlsx` 형식으로 고정되어 있습니다.

**Q. 결과 저장 폴더를 바꿀 수 있나요?**
A. 현재 버전은 `C:\Temp\query-excel-download\` 로 고정되어 있습니다.

**Q. DB 접속에 실패합니다.**
A. DB 추가 화면의 **[연결 테스트]** 로 원인을 먼저 확인하세요. JDBC URL,
계정/비밀번호, 네트워크(방화벽), DB 가동 여부를 점검합니다.

**Q. 날짜 조건이 결과와 맞지 않습니다.**
A. 쿼리 저장 시 지정한 **날짜포맷**(`yyyyMMdd` / `yyyy-MM-dd`)이 DB의 날짜
저장 형태와 일치하는지 확인하세요.
